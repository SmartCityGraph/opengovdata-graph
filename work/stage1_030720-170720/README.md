### Предварительное предложение по заполнению индивидов онтологии
В случае, если мы оставим только один классфикатор предлагаю именовать объект в формате его полного кода, в случае, если оставим оба, перед номером будет уточнение классификатора «ОКАТО_» или «ОКТМО_». 

Тип муниципального образования необходимо каким-либо образом определять. Делать это будем с помощью концептов.

hasNestedObject и includedIn – object property используются для построения иерархии среди индивидов ("имеет внутренний муниципалитет" и "входит в" соответственно).
Skos:definition – определение
Skos:prefLabel – официальное название
skos:editorialNote - дополнительная информация
hasAbbr - рассшифровка аббревиатуры
hasDate - дата
	hasDateOfIntroduction - дата введения
	hasDateOfAccept - дата принятия
hasCode - код
	hasFullCode - полный код
	hasValueCode - значимая часть
hasChange
	hasChangeType - тип изменения
	hasChangeNumber - номер изменения
hasDescription - описание
hasCapital – столица, если есть
